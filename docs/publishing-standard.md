# Chopsticks Guide 发文标准

> 适用范围：筷子站所有 guides、materials、FAQ、后续新增文章页。  
> 目标：保证批量发文时结构统一、可读性稳定、内链自然、SEO/GEO 可控。

## 1. 发文前必须具备的信息

每篇文章生成前必须先准备：

| 字段 | 标准 |
|---|---|
| Primary Keyword | 一个主关键词，不与已发布页面冲突 |
| Search Intent | informational / comparison / commercial-lite / etiquette / tutorial |
| URL Slug | 英文小写短横线，不堆词 |
| SEO Title | 35-70 字符，核心词尽量靠前 |
| Meta Description | 90-165 字符，说明用户能得到什么 |
| Article Type | guide / comparison / tutorial / etiquette / material / FAQ |
| Target Page Role | 枢纽页 / 长尾页 / 工具支撑页 / 购买前引导页 |
| Related Internal Links | 3-5 个候选内链 |
| FAQ Plan | 2-5 个当前页面专属 FAQ |

## 2. 写作原则

### 2.1 核心原则

- 先回答，再解释。
- 以使用场景为中心，不写空泛文化散文。
- 礼仪内容写成 cultural guidance，不写成绝对规则。
- 材质和购买内容写成 comparison guidance，不写成产品承诺。
- 所有内容都要能服务真实搜索意图。

### 2.2 开头规则

开头 80-140 个英文词内必须完成三件事：

1. 直接回答搜索问题。
2. 说明这个页面解决什么实际困惑。
3. 告诉用户为什么继续看下去。

不要这样写：

```text
Chopsticks have a long and fascinating history. In this article, we will explore everything you need to know.
```

### 2.3 过渡规则

每个 H2 之间要有自然过渡，说明“为什么下一个部分值得看”。

推荐写法：

- “That matters because grip changes how easy the chopsticks feel in daily use.”
- “Once the basic hold is clear, the next problem is usually food control.”
- “Material matters less than grip at the start, so compare texture before decoration.”

避免：

- Firstly / Secondly / Finally
- Moreover / Furthermore 连续堆叠
- 没有解释就突然跳主题

### 2.4 正文节奏

| 项目 | 标准 |
|---|---|
| 段落长度 | 普通段落 2-4 句 |
| 句子节奏 | 长短句搭配，核心答案用短句 |
| 表格使用 | 材质对比、优缺点、使用场景优先用表格 |
| 列表使用 | 步骤、误区、选择建议优先用列表 |
| GEO 摘取 | 每篇至少有一个可直接引用的短答案块 |

推荐正文结构：

```text
直接答案
-> 为什么这样判断
-> 步骤 / 对比 / 表格
-> 常见误区或礼仪说明
-> Related Guides
-> FAQ
-> 结尾
```

### 2.5 结尾规则

结尾要完成三件事：

1. 回答开头的问题。
2. 重申最重要的判断标准。
3. 自然引导到下一个页面。

推荐写法：

```text
The main point is simple: beginners usually improve faster when grip and control come before style. Once you know how the chopsticks should move, you can compare materials, learn etiquette, or choose a better pair for daily use.
```

不要这样写：

```text
In conclusion, we hope this article helped you. Thanks for reading.
```

## 3. SEO/GEO 结构标准

### 3.0 Content depth gate

Do not publish a new chopsticks article as a short placeholder.

| Page type | Minimum visible words | Target words |
|---|---:|---:|
| New SEO/GEO guide | 800 | 900-1200 |
| Product or buying guide | 800 | 1000-1400 |
| Material comparison page | 700 | 900-1200 |
| Tool/list/FAQ support page | Not forced | Judge by task completeness |

Every new article must include a direct answer block, use-case explanation, comparison or checklist content, common mistakes, related guides, and page-specific FAQ. Existing short pages should be treated as rewrite candidates, not as the quality benchmark.

| 项目 | 标准 |
|---|---|
| H1 | 每页仅 1 个 |
| H2 | 至少 2 个，结构清晰 |
| FAQ | 详情页 2-5 个专属 FAQ |
| FAQ Schema | 每页只输出 1 份 FAQPage |
| Breadcrumb | 必须有 |
| Canonical | 必须指向当前正式 URL |
| Sitemap | 发布后进入 sitemap |
| 内链 | 每篇 3-5 条，自然放入正文或 Related Guides |
| 图片 Alt | 所有非装饰图必须有 alt |
| GEO 短答案 | 页面内至少 1 段可直接引用 |

## 4. 内链规则

优先链接到：

1. 上级枢纽页，如 `/guides/`
2. 对应专题页，如 `/how-to-use-chopsticks/`
3. 材质页或比较页，如 `/materials/chopstick-material-compare/`
4. 与当前问题强相关的辅助页，如 `/guides/how-to-eat-rice-with-chopsticks/`

禁止：

- 为了数量硬塞内链
- 每篇文章都放一模一样的 Related Guides
- 锚文本全部写成 click here

## 5. FAQ 规则

FAQ 必须满足：

- 每页 2-5 个
- 问题像真实用户会问的话
- 答案短、直接、可引用
- 与当前页面直接相关，不要全站复制

不建议写：

- 绝对化礼仪判断
- 产品安全承诺
- 医疗、营养、法律结论

## 6. 图片和视觉规则

| 项目 | 标准 |
|---|---|
| 视觉类型 | 筷子插画、握法示意、餐桌元素、简洁传统纹样 |
| 图片数量 | 普通文章 1-3 张即可 |
| 图片相关性 | 必须和握法、材质、礼仪或场景相关 |
| 禁止 | 无关 stock 图、模糊背景图、看不清文字的图 |
| Alt | 描述真实内容，不堆关键词 |

## 7. 合规边界

- 不把礼仪写成文化优越判断。
- 不把材质比较写成绝对推荐。
- 不写产品功效承诺。
- 不写“must buy”“guaranteed best”这类绝对表达。
- 不碰医疗、儿童安全、食品安全的硬性专业承诺。

推荐替代表达：

| 不建议 | 建议 |
|---|---|
| best forever | usually easier for beginners |
| guaranteed non-slip | often offers more grip |
| wrong in every case | commonly avoided in formal settings |
| safest choice | a practical starting option |

## 8. 发布前检查

```bash
npm run build
npm run audit
npm run test:tools
```

检查门槛：

- `Fix = 0`
- 无内部 404
- FAQ、canonical、breadcrumb 正常
- 新文章页有搜索框、Related Guides、FAQ
